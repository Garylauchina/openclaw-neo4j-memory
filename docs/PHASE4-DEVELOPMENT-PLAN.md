## 📝 开发流程

### 1. 认领任务
- Agent 在对应 Issue 下评论认领
- 技术负责人分配任务

### 2. 开发
- 创建分支：`fix/issue-#30-简短描述` 或 `feature/任务描述`
- 编码并本地测试
- 确保语法验证通过

### 3. 提交 PR
- PR 标题：`fix: 描述` 或 `feat: 描述`
- 关联 Issue：`Closes #30`
- 等待技术负责人审核

### 4. 审核标准
| 维度 | 要求 |
|------|------|
| 代码质量 | 无语法错误，符合现有代码风格 |
| 测试覆盖 | 核心逻辑有测试用例 |
| 文档更新 | 涉及 API 变更需更新文档 |
| 性能影响 | 无明显性能退化 |

### 5. 合并与发布
- 技术负责人审核通过
- 合并到 `main` 分支
- 发布新版本 tag
