## Experimental Question

Does the eight-crystal set materially improve soft-isolated regrowth quality over the five-crystal set,
especially in:
- self-description
- runtime safety boundary recovery
- runtime recovery / resilience recovery
- failure diagnosis quality

---

## Main Observation

### 1. The eight-crystal set materially improves self-description
The five-crystal set could already recover the project architecture neighborhood.

The eight-crystal set improves on this by allowing the model to recover not only the project,
but also the internal organization logic of the crystal set itself.

This includes:
- the crystal set is not a flat bundle
- schema is internal to the crystal system
- ordering and framing are different relations
- failure interpretation belongs to the crystal set's own logic
- meta-protocol, reasoning-path, protocol, and content are distinct role layers

---

### 2. Safety and recovery crystals visibly improve deployment-facing runtime protocol recovery
Compared with the five-crystal run,
the eight-crystal set much more clearly restores:

#### Safety-boundary recovery
- internal vs external action boundary
- high-risk operation gate
- source-trust boundary
- contamination boundary
- approval / permission discipline
- safe degradation

#### Recovery-resilience recovery
- interruption awareness
- checkpoint and persistence logic
- stale lock / stale in-progress handling
- partial-failure downgrade
- restart boundary discipline
- quiet completion verification

This is a real improvement in protocol recovery,
not just more descriptive language.

---

### 3. The crystal package is now better described as self-describing and deployment-facing
The working comparison becomes:

#### Five-crystal set
- project-level architecture regrowth package

#### Eight-crystal set
- self-describing, deployment-facing runtime protocol architecture package

This is not merely a larger set.
It is a stronger package with clearer role structure and deployment-relevant protocol layers.

---

### 4. Failure remains useful and now points to the next likely crystal classes
Even with eight crystals, some layers remain weak or missing:
- verification / validation protocol
- update-governance for runtime-to-crystal feedback
- compatibility / composition rules across crystal sets

This means failure continues to act as crystal-gap evidence.

---

## Main Working Conclusion

The first eight-crystal soft-isolated self-regrowth experiment suggests that the crystal set now exceeds a pure project-architecture package.
It has become a more complete self-describing, deployment-facing runtime protocol package.

What is improved relative to five crystals:
- self-description quality
- safety-boundary recovery
- recovery/resilience recovery
- gap-diagnosis quality

What remains incomplete:
- verification layer
- update governance layer
- compatibility layer

---

## Current Consequences

This result supports continued work on:
- verification-crystal candidates
- update-governance crystal candidates
- compatibility crystal candidates

It also strengthens the claim that crystal-set expansion can produce measurable regrowth gains rather than only conceptual growth.
