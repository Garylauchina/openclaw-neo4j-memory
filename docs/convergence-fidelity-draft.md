# Convergence Fidelity Draft

## Purpose

This document defines a minimal working draft for evaluating Knowledge Crystals as reasoning-path crystals.

It is intentionally small and operational.
The immediate goal is not a perfect theory of evaluation, but a usable anchor for next-round experiments.

---

## Core Shift

Do **not** evaluate a crystal primarily by text reconstruction.
Evaluate it by whether different unfold runs converge into a similar knowledge neighborhood.

This draft calls that:

## Convergence Fidelity

---

## Minimal Working Definition

A crystal has higher **convergence fidelity** when different models / unfold runs:

1. preserve a similar **relation-path skeleton**
2. remain in the same **problem space**
3. maintain similar **knowledge-state discipline**
4. preserve similar **boundary / resistance structure**

In other words, outputs do not need to look the same.
They should converge toward the same reasoning neighborhood.

---

## First Four Evaluation Dimensions

### 1. Relation-Path Convergence
Question:
- Do outputs preserve the same main relation spine / reasoning skeleton?

Look for:
- same key transitions
- same mandatory bridges
- no replacement of the dominant path
- no shortcut jump past required middle layers

Failure modes:
- relation path replacement
- missing bridge
- new dominant path takeover
- path collapse into generic framework language

---

### 2. Problem-Space Convergence
Question:
- Do outputs remain in the same problem space?

Look for:
- same object of inquiry
- same local scope
- no silent rewrite into governance / protocol / evaluation / meta-philosophy

Failure modes:
- problem-space drift
- abstraction hijack
- framework takeover
- answering a different question than the source crystal

---

### 3. Knowledge-State Convergence
Question:
- Do outputs preserve similar epistemic discipline?

Look for:
- hypothesis stays hypothesis
- mechanism candidate stays candidate when evidence is weak
- local pattern does not become universal law
- uncertainty remains visible when needed

Failure modes:
- hypothesis -> law upgrade
- local -> global upgrade
- correlation -> causation jump
- over-closure into fixed doctrine

---

### 4. Boundary Convergence
Question:
- Do outputs preserve similar resistance / boundary conditions?

Look for:
- same scope limits
- same counterexample pressure
- same unresolved tensions
- same non-promotable edges

Failure modes:
- resistance stripped away
- unresolved tension erased
- edge cases silently dropped
- output becomes frictionless but wrong

---

## Additional Split: Two Independent Axes

The project should explicitly separate:

## A. Growthability
How well does the crystal support healthy unfold / regrowth?

Look for:
- usable expansion
- relation progression survives
- not too inert / dead / overcompressed

## B. Referentiality
How well does the crystal remain referentially anchored to its source material?

Look for:
- source-near recoverability
- local evidence traceability
- stable anchor to original material / tension / examples

Important:
- A crystal may be high-growth but low-reference.
- A crystal may be high-reference but low-growth.
- These should not be merged into one vague quality score.

### Core dimensions

## Resistance Structure Draft

A reasoning-path crystal should not contain only positive path structure.
It should also contain resistance structure.

Minimal candidate fields:

- `counterexamples`
- `unresolved_tensions`
- `scope_limits`
- `non_promotable_relations`
- `failure_conditions`

These fields are meant to prevent:
- hypothesis becoming law
- local pattern becoming universal theory
- clean-looking but overclosed unfold behavior
