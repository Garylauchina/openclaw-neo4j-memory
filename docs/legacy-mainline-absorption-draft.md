# Legacy Mainline Absorption Draft

## Purpose

This draft prevents the architecture refactor from becoming only a narrative rewrite.

Its goal is to absorb the old mainline into the new four-stage project mainline by answering:
- what already exists
- what is partly implemented
- what is experimentally validated
- what remains conceptual
- where each asset belongs in the new architecture

The new mainline is:

## memory -> experience -> knowledge crystal -> Agent-to-Agent transfer

This document makes sure existing project assets are not lost under the new framing.

---

## Status Labels

Use the following working labels:

- **Implemented**: running code or stable operational behavior exists
- **Partial**: some real implementation exists, but incomplete or not yet integrated
- **Experimentally validated**: supported by experiments, but not fully operationalized
- **Conceptual**: present mainly in issue/docs/research framing
- **Historical reference**: useful prior work, but not current mainline center

---

## Practical Reframe

The architecture refactor should therefore be read not as:
- replacing the old project with a new idea

but as:

## reindexing existing assets into a clearer mainline

That means:
- foundational storage work stays
- meditation / reflection work stays
- metacognitive and governance work stays
- raw-reflection results stay
- crystal work becomes more explicit
- transfer work becomes a new top layer

---

## Current Risk to Avoid

The main risk in the refactor is:

## narrative overwrite

That would happen if:
- new crystal/transfer language hides old implemented achievements
- old experience-layer assets are treated as obsolete just because the mainline vocabulary changed
- validated experimental branches get flattened into vague architectural slogans

This draft exists to prevent that.

---

## One-Sentence Summary

The architecture refactor should absorb and reorganize the old mainline,
not replace it.

The old work already contains foundational assets for:
- memory
- experience
- crystal formation
- transfer thinking

The task now is to make those assets legible inside the new mainline.
