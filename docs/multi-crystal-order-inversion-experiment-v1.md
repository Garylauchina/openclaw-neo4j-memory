# Multi-Crystal Order Inversion Experiment v1

## Purpose

This document records the order-inversion experiment for the first multi-crystal runtime.

The central question is not simply whether multi-crystal is better than single-crystal.
It is:

## does the methodology crystal matter because of its content,
## or because of its protocol position?

To test this, the reading order was inverted.

---

## Comparison

### Baseline order
- methodology crystal first
- system crystal second

### Inverted order
- system crystal first
- methodology crystal second

Model used:
- `openai-codex/gpt-5.4`
- prompt-fed self-experiment

---

## Experimental Question

When the order is inverted:
- does the methodology crystal still shape the run in the same way?
- or does it become a later correction on an already formed system interpretation?

---

## Main Observation

### 1. Reading the system crystal first changes the agent's initial stance
When the system crystal is read first, the model naturally enters:
- system reconstruction mode
- architecture regrowth mode
- tension-as-architecture-tradeoff mode

This happens before the methodology crystal arrives.

That means the methodology crystal does not begin from a neutral state.
It begins from an already-shaped reading posture.

---

### 2. The methodology crystal still works, but in a weaker way
In inverted order, the methodology crystal does not disappear.
It still adds:
- calibration language
- anti-hardcoding boundaries
- scope control
- adaptation awareness

But its role changes.

Instead of acting as:
- entrance shaping

it becomes:
- mid-course correction

This difference is substantial.

---

### 3. The four-layer separation becomes weaker under inversion
In the baseline multi-crystal experiment, the model more clearly separated:
- methodology constraints
- system structural content
- calibration results
- adaptation / reconstruction policy

Under inversion, these layers are still present,
but they become more entangled because the system crystal takes interpretive priority first.

---

## Main Conclusion

The order-inversion experiment supports the following working conclusion:

## methodology crystal is not only useful because of what it says,
## but because of where it appears in the runtime protocol.

This means:

## crystal sets are not flat bundles
## they are protocol objects with ingestion order

---

## Stronger Protocol Reading

At the current stage, the stronger runtime reading is:

## methodology crystal first
## system crystal second
## regrowth third

Why:
- method should shape entry into structure
- calibration should precede system interpretation
- adaptation should be derived before regrowth begins

The inverted order does not fully fail,
but it weakens the role of the methodology crystal by making it corrective rather than foundational.

---

## Architectural Consequence

Crystal-set schema will likely need explicit fields for:
- ingestion order
- dependency structure
- precondition relationships

Possible future examples:
- `must_be_read_before`
- `depends_on`
- `calibrates_entry_into`
- `requires_prior_methodology`

This suggests crystal sets are better understood as:

## cognitive protocol object sets

rather than only grouped content packages.

---

## Current One-Sentence Summary

The order-inversion experiment suggests that methodology crystals work not only by providing content,
but by occupying the correct protocol position before system crystals are interpreted.
