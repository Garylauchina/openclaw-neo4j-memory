## 📊 Neo4j 图谱大修复（下午至晚上）

### 发现的问题
- MERGE 键用了 `{name, entity_type}` → 同一名称不同类型创建重复节点
- 缺少质量过滤 → 垃圾实体（"的""了""一下"）混图谱
- 冥思批处理限制太小（15-30）→ 处理效率极低

### 执行的修复（4项）
1. **MERGE 改用 name 单键** → 消除重复节点
2. **新增质量过滤器** → 100+ 停用词 + 规则过滤碎片实体
3. **批处理限制提升** → 15/20 → 100/500（吞吐量 3-5x）
4. **蒸馏参数调整** → min_cluster_size: 5→3, max_meta_nodes: 10→50

## 🦌 DeerFlow 部署到 kakamac-mini（22:40 → 23:10）

### 项目
**bytedance/deer-flow** — 字节跳动开源的 SuperAgent 框架，支持子代理、记忆、沙盒、多 IM 频道（Telegram/Slack/Feishu等）

### 目标机器
- **主机**: kakamac-mini (kaka@kakamac-mini)
- **架构**: Apple Silicon (arm64, T6041)
- **已有环境**: Ollama + gemma4:e4b 本地模型 (localhost:11434/v1, 9.6GB)
- **限制**: 无 sudo 权限、无 Homebrew、无 Docker、无 Node.js

### 安装过程
1. **NVM** → git clone nvm 到 ~/.nvm
2. **Node.js 22** → nvm install 22 (v22.22.2)
3. **pnpm** → npm install -g pnpm (v10.33.0)
4. **uv** → curl 安装 (v0.11.3)
5. **克隆仓库**: ~/deer-flow (git clone)
6. **配置 .env**: OPENAI_API_KEY=ollama + TELEGRAM_BOT_TOKEN
7. **配置 config.yaml**: 添加 Ollama gemma4:e4b 模型 + Telegram channel
8. **安装依赖**: make install (Python uv + pnpm frontend)

### 遇到的挑战
- **nginx 依赖**: `make check` 要求 nginx，但无法安装（缺sudo）
- **解决方案**: patch `scripts/check.py` + `scripts/serve.sh` 移除 nginx 检查/启动
  - nginx 只是反向代理(port 2026)，IM 频道直接走 long-polling 不需要它
