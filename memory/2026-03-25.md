1. **表达长期认知**：通过State Vector（权重、置信度、证据数）记录

#### ✅ **确定性更新实现**
1. **Merge公式**：`new_weight = α * old_weight + (1 - α) * delta`（α=0.8）
2. **时间衰减**：`weight *= exp(-λ * time_diff)`（λ=0.01）
3. **冲突处理**：保留两条边，不强制合并

```python
class GlobalGraph:

    def create_node(self, name: str, node_type: NodeType) -> str

    def merge_node(self, node_a: str, node_b: str, confidence: float = 0.8) -> bool

```

#### **数据结构**

4. **Diff**：op, src, dst, type, delta, confidence, node_a, node_b

## 🚀 **Active Set（多子图调度系统）实现**

### 1. 实现时间
**开始时间：** 2026-03-25 17:07 (接收完整规范)
**完成时间：** 2026-03-25 17:25 (18分钟完成实现)
**实现依据：** 用户提供的完整多子图调度系统规范

### 2. 系统升级本质
**从：** Memory System (记忆系统)
**进入：** ❗ Cognitive Routing System (认知路由系统)

**当前阶段：** Subgraph-level Attention System
**类比Transformer：** token attention → subgraph attention

### 3. 核心问题解决
#### ✅ **单子图问题解决**
1. **话题切换"断层"问题** → 多子图缓存和复用
2. **信息强制混合问题** → 加权融合上下文
3. **无法形成"长期思维线程"** → 子图状态管理和生命周期

### 4. 核心模块实现
#### ✅ **三个核心函数**
```python
1. get_candidate_subgraphs(query) → List[SubgraphState]
2. score_subgraph(S, query) → float
3. build_active_set(query) → ActiveSet
```

#### ✅ **Active Set定义**
```python
ActiveSet = {
    S1: {subgraph, weight},
    S2: {subgraph, weight},
    ...
}
```

### 5. 构建规则实现
#### ✅ **Step 1: 候选子图来源**
1. **当前新构建子图**：`S_new = build_active_subgraph(query)`
2. **历史子图（缓存）**：`S_cache = retrieve_from_cache()`
3. **合并**：`candidates = [S_new] + S_cache`

#### ✅ **Step 2: 子图打分（核心）**
**评分函数（统一实现）：**
```python
score(S) = 
    0.5 * relevance_to_query +      # 相关性（最重要）
    0.3 * recency +                 # 最近性
    0.2 * internal_coherence        # 结构质量
```

**具体实现：**
1. **relevance**：`embedding(query) vs embedding(subgraph)` (简单话题匹配)
2. **recency**：`exp(-λ * time_since_last_use)` (λ=0.01)
3. **coherence**：`avg(edge.weight)` (平均边权重)

#### ✅ **Step 3: 选择Top-K子图**
```python
ActiveSet = top_k(candidates, k=3~5)
强约束：max_active_subgraphs ≤ 5
```

#### ✅ **Step 4: 注意力分配（关键）**
```python
Softmax分配：weights = softmax(scores)
输出：[(S1, 0.6), (S2, 0.3), (S3, 0.1)]
```

#### ✅ **Step 5: 上下文融合（最关键一步）**
**升级前：** 单子图 → context
**升级后：** 多子图 → 加权融合 → context

**融合策略（第一版简单版）：**
```python
context = []
for S, w in ActiveSet:
    context.append(project(S, weight=w))
```

**输出格式：**
```
[主要话题 - 权重0.6]
...

[次要话题 - 权重0.3]
...
```

### 6. 缓存机制升级
**升级前：** `cache_key = topic + intent`
**升级后：** `cache_key = subgraph_signature`
